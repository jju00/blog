# 개념

---
침투 테스트나 워게임 환경에서 공격자가 대상 시스템에 명령 실행 권한을 확보하기 위해 사용하는 방식에는 여러 형태의 **쉘(shell)** 과 **페이로드(payload)** 가 존재한다.  
각 방식은 연결 방향, 네트워크 환경(방화벽), 흔적 여부, 운영 편의성 등에 따라 장단점이 다르다.

- **바인드 쉘 (Bind Shell)**  
    대상 시스템이 특정 포트를 열고 공격자의 접속을 기다리는 방식
    
- **리버스 쉘 (Reverse Shell)**  
    대상 시스템이 공격자 시스템으로 먼저 연결을 시도하여 세션을 생성하는 방식
    
- **웹 쉘 (Web Shell)**  
    웹 서버에 업로드된 스크립트를 통해 웹 요청으로 명령을 실행하는 방식
    
- **페이로드 (Payload)**  
    쉘 동작을 수행하기 위해 전달·실행되는 실제 코드로,  
    **전달 방식(staged / stageless)**, **플랫폼**, **언어**, **쉘 종류**의 조합으로 구성된다.


## 바인드 쉘

> _**바인드 쉘**_: 대상 호스트의 특정 포트에서 공격자의 접속 요청을 기다리는 쉘 유형

![[Pasted image 20250622215303.png]]
1. 바인드 쉘이 업로드/실행
2. 프로세스가 대상 호스트에서 실행되게 되며, 특정 포트 번호에서 쉘을 열고 대기 상태에 들어감
3. 이후 공격자는 바인드 쉘이 "묶인(bind)" 포트를 찾아가 쉘에 접속하고 추가적인 명령이나 페이로드를 전달
4. 그 결과를 전달받음

## **리버스 쉘**

> _**리버스 쉘**_: 타겟 호스트에서 공격자 호스트의 특정 포트로 ***먼저 연결을 시도***한 뒤 세션을 구축하고, 공격자에게 접근 권한을 제공하는 쉘의 유형

- 바인드 쉘이 대상 호스트의 특정 포트에 묶여(bind) 공격자를 기다리는 것과는 반대로, 리버스 쉘은 대상 호스트에서 먼저 공격자 호스트의 특정 포트로 연결을 시작한다.
![[Pasted image 20250622215348.png]]
1. 리버스 쉘 업로드/실행
2. 공격자 서버로 리버스 쉘 채널/세션 구축(리버스 쉘이 먼저 공격자에게 connection을 시도)
3. 공격자가 그 채널/세션으로 명령을 전달
4. 그 결과를 전달 받음

### 특징

> ‘방화벽’이 가장 큰 장점이다. 보통 서버는 특정 포트만 열어놓는 경우가 많음
> 
> - 바인드 쉘이 포트 7777에 열어서 7777을 열어도 방화벽 때문에 7777로 갈 수 있는 방법이 없다.
> - 리버스 쉘은 ***‘서버가 공격자에게 먼저 연결을 시도’*** 하는 거기 때문에 어떻게든 connection이 될 확률이 크다.![[Pasted image 20250622215404.png]]

## 웹 쉘

> _**웹 쉘**_: 공격자가 웹 서버에 업로드하고 웹 브라우저를 통해 원격으로 명령을 실행할 수 있게 하는 페이로드의 유형

- 웹 서버가 실행 가능한 악성 스크립트 및 파일 (php, jsp, asp, cgi/sh, 등) 의 형태로 웹 서버나 웹 앱을 통해 명령을 실행한다.
![[Pasted image 20250622215516.png]]
대충 이렇게 생김

### 특징

>웹 쉘은 대상 호스트 파일 시스템 위에 흔적을 남기고, 누구나 접근이 가능하다
>⇒ 따라서 비밀번호로 접근이 가능한 웹 쉘을 사용
>⇒ 물론 약한 비밀번호나 잘못 설정된 경우 허가 받지 않은 공격자들이 웹 쉘을 사용할 수 있다. 결론적으로, 웹 쉘 사용은 최대한 지양한다.


## 페이로드

- **페이로드 전달 방식 유형**
    - ==**staged**==: 필요한 페이로드를 공격자 서버로부터 추가로 다운받아 실행
    - ==**stageless**==: 필요한 페이로드가 모두 페이로드 자체내에 포함되어 있음
    
- **페이로드의 구성 - 대상 + 포멧(페이로드의) + 전달 방식 + 종류**
    - 리눅스 os 기반 + C언어로 컴파일된 ELF + Staged + 리버스 쉘
    - 웹 서버 + PHP언어로 만들어진 + Stageless + 웹 쉘
    - 웹 서버 + PHP언어로 만들어진 + Staged + 리버스 쉘
    - Windows os 기반 + Powershell로 만들어진 + Stageless + 바인드 쉘
    - 등등
